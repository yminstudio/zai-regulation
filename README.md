# 사규 AI 검색 챗봇 개발 계획서

- **작성일:** 26.02.27
- **작성자:** 오하늘
- **버전:** 1

## 사내 규정 검색 기반 챗봇

- **목적:** 사내 규정(사규)을 검색 기반으로 조회하고, 근거 링크 포함 답변을 제공하는 챗봇 구축
- **사용 대상:** 인사팀(우선) → 전사 확장
- **최종 산출물:** Open WebUI에서 사규 모델로 채팅 가능 + 운영/갱신 체계 구축

## 개발 추진 전략

인사팀과 논의하여, 해당 프로젝트의 실효성 검증 및 개발 기획 고정.

- **1차 서비스 개발:** OpenAI 기반으로 빠르게 안정화(품질 확보) → 사내 LLM(Ollama) 전환 가능 구조 확보(Provider Adapter)
- **네트워크 구조:**
  - 125(WebUI/Ollama) → 166(APP) 호출은 68 경유
  - 166(APP) → 125(Ollama) 호출은 직접 가능
- **2차 서비스 개발:** 인사팀 및 사내 인원들 피드백 반영. 플랫폼 운영 설계.

## 인프라 구조도 (실제 사내 LLM 활용 시)

| 서버 | 역할 |
|------|------|
| **125 (H200)** | Open WebUI, Ollama(사내 LLM) |
| **166 (내부망)** | Weaviate(Vector DB), Ingestion 파이프라인, APP(FastAPI) |
| **68** | HTTPS 종단 / Reverse Proxy(125→166 통신 경유) |
